# SECURITY

## 매매 차단 원칙 (코드 미정의가 유일 방어선)

본 MCP 서버는 **시장 데이터 분석 전용**입니다. 사용자 자금이 움직이는 어떠한 동작도 수행하지 않습니다.

### 한투 API에는 read-only 권한이 없다

먼저 알아두어야 할 사실: **한국투자증권 OpenAPI는 조회 전용(read-only) 권한이라는 옵션 자체를 제공하지 않습니다.** 사용자가 신청서에서 "조회만" 같은 항목을 선택할 수 있는 길이 없습니다. 따라서 발급된 한투 API 키는 본질적으로 매매 권한을 포함합니다.

이 사실 때문에 "API 키 권한으로 매매를 막는" 1차 방어선은 **존재할 수 없으며**, 본 MCP의 매매 차단은 다음 단일 방어선에 전적으로 의존합니다.

### 방어선: 코드에서 매매 엔드포인트를 처음부터 정의하지 않음

이것이 본 프로젝트가 한투 공식 샘플을 그대로 포팅하지 않고 TypeScript로 처음부터 다시 짠 핵심 이유입니다. 매매 관련 코드를 단 한 줄도 작성하지 않았습니다.

- `src/kis/endpoints.ts` 에는 `/uapi/domestic-stock/v1/trading/*` 패턴이 **존재하지 않습니다**.
- `src/tools/` 어디에도 주문/취소/잔고/예수금 조회 도구가 정의되어 있지 않습니다.
- MCP 서버는 노출하지 않은 도구는 LLM이 호출할 수 없습니다. 즉 LLM이 아무리 "삼성전자 100주 사줘"라고 요청을 받아도, 호출할 도구가 없으므로 동작할 수 없습니다.
- 검증:
  ```bash
  grep -r -E "trading|order|inquire-balance|inquire-psbl-order" src/
  # → 결과 0건이 정상
  ```

방어선이 코드 하나뿐이라는 사실은 운영 측면에서 다음을 의미합니다:
- 본 저장소를 fork하여 매매 엔드포인트를 추가하면 매매가 가능해집니다. 그것은 더 이상 본 프로젝트가 아닙니다.
- 의존성 업데이트 또는 PR 머지 시 새로운 매매 도구가 추가되지 않았는지 위 grep 명령으로 매번 확인하십시오.

## 토큰 / 시크릿 보호

- 토큰은 Cloudflare KV에만 저장됩니다 (TTL 23시간, 만료 1시간 전 갱신).
- 토큰, `KIS_APP_KEY`, `KIS_APP_SECRET` 은 어떤 응답·로그·에러 메시지에도 포함되지 않습니다.
- 토큰 발급 실패 시 응답 본문에 포함될 수 있는 `Bearer ...` 패턴은 출력 전 마스킹됩니다 (`src/kis/auth.ts`).
- 시크릿은 `wrangler secret put` 으로만 등록하며, `.dev.vars` / `.env` 는 git 커밋 금지 (`.gitignore` 등록 완료).

## 레이트 리미팅

- 한투 공식 제한: 초당 20회 (실전).
- 본 MCP는 자체 한도 **초당 15회** (안전마진).
- ETF 신용거래율 집계처럼 다수 호출이 필요한 메타 도구도 동일 한도를 따릅니다.
- 429 / `EGW00201` 응답 시 지수 백오프(250ms × 2^n)로 최대 3회 재시도.

## 개인 계좌 정보 비접근 원칙

- 잔고, 보유종목, 매수가능금액, 주문내역 등 **개인 계좌에 귀속된 데이터를 조회하는 도구가 없습니다.**
- 본 MCP는 시장 전체에 대한 공개 데이터(시세, 랭킹, 구성종목, 공매도/대차 통계)만 다룹니다.
