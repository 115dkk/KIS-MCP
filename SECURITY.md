# SECURITY

## 매매 차단 원칙 (이중 방어)

본 MCP 서버는 **시장 데이터 분석 전용**입니다. 사용자 자금이 움직이는 어떠한 동작도 수행하지 않습니다. 두 단계 방어가 적용됩니다:

### 1차 방어: 한투 API 키 권한
- 한국투자증권 OpenAPI 포털에서 **조회 전용 권한 키**로 발급받아 사용합니다.
- 매매·잔고·주문 권한이 포함된 키는 **사용 금지**.
- 키 발급 시 신청서에서 다음 권한이 **포함되지 않았는지** 반드시 확인:
  - 주식주문(현금/신용)
  - 주식정정/취소주문
  - 주식체결/잔고/예수금/매수가능 조회
  - 매도가능 조회

### 2차 방어: 코드에서 매매 엔드포인트 미정의
- `src/kis/endpoints.ts` 에는 `/uapi/domestic-stock/v1/trading/*` 패턴이 **존재하지 않습니다**.
- `src/tools/` 어디에도 주문/취소/잔고 도구가 정의되어 있지 않습니다.
- 검증:
  ```bash
  grep -r -E "trading|order|inquire-balance|inquire-psbl-order" src/
  # → 결과 0건이 정상
  ```

## 토큰 / 시크릿 보호

- 토큰은 Cloudflare KV에만 저장됩니다 (TTL 23시간, 만료 1시간 전 갱신).
- 토큰, `KIS_APP_KEY`, `KIS_APP_SECRET` 은 어떤 응답·로그·에러 메시지에도 포함되지 않습니다.
- 토큰 발급 실패 시 응답 본문에 포함될 수 있는 `Bearer ...` 패턴은 출력 전 마스킹됩니다 (`src/kis/auth.ts`).
- 시크릿은 `wrangler secret put` 으로만 등록하며, `.dev.vars` / `.env` 는 git 커밋 금지 (`.gitignore` 등록 완료).

## 레이트 리미팅

- 한투 공식 제한: 초당 20회 (실전).
- 본 MCP는 자체 한도 **초당 15회** (안전마진).
- ETF 신용거래율 집계처럼 다수 호출이 필요한 메타 도구도 동일 한도를 따릅니다.
- 429 / `EGW00201` 응답 시 지수 백오프(250ms × 2^n)로 최대 3회 재시도.

## 개인 계좌 정보 비접근 원칙

- 잔고, 보유종목, 매수가능금액, 주문내역 등 **개인 계좌에 귀속된 데이터를 조회하는 도구가 없습니다.**
- 본 MCP는 시장 전체에 대한 공개 데이터(시세, 랭킹, 구성종목, 공매도/대차 통계)만 다룹니다.
